01 · Introduction

Problem Statement & Research Scope

The rapid proliferation of Fifth Generation (5G) mobile network infrastructure has fundamentally altered the threat landscape for telecommunications security. As 5G assumes a load-bearing role across latency-sensitive applications — autonomous vehicular systems, industrial automation, smart city deployments — network reliability and the timely detection of anomalous behaviour become imperatives of critical importance.

Anomalous conditions in 5G traffic may manifest as security intrusions, performance degradation, or hardware failures. The protocol heterogeneity and pervasive end-to-end encryption characteristic of 5G environments render real-time detection substantially more difficult than in preceding generations.

This work presents a systematic evaluation of multiple anomaly detection paradigms applied to 5G traffic: classical supervised and unsupervised ML, deep residual network architectures, sequence-based n-gram modelling, GAN-based data augmentation, and a federated learning framework enabling collaborative detection without centralised data exposure. Experiments are conducted on two publicly available 5G traffic corpora — 5G-NIDD and 5G-PFCP.

Evaluation & Explainability

Feature engineering

Features are extracted at the session level (5–10 min windows) across four categories:

  • Temporal features — per-pair inter-arrival times (mean, median, std), burst count, mean & max burst length
  • Packet-size statistics — total, incoming, outgoing counts; mean, std, median per direction; size & direction of first 10 packets
  • Protocol & header fields — port numbers, protocol ID, control flags (SYN, ACK...)
  • Timestamp-derived features — hour, minute, second encoding diurnal and sub-minute patterns

04 · Evaluation Protocol

Three Experimental Scenarios

Models are evaluated under three dataset construction scenarios via k-fold cross-validation (k = 3).

Zero-day robustness

Scenario A — Benign-only training

Training contains only normal traffic; test set has unseen attacks. Highest memory overhead (3,072 MB). Evaluates zero-day generalisation.

Partial attack knowledge

Scenario B — One attack type

Training includes one known attack class + benign. Longest runtime (4,215 s), reflecting cross-distribution generalisation difficulty.

Broad attack coverage

Scenario C — Four attack types

Training covers four attack families + benign. Best runtime–memory trade-off; closest approximation to production deployment.

07 · Deep Learning

ResNet18 for Network Flow Classification

ResNet18 is adapted for traffic classification with 18 layers across four stages, each containing two convolutional blocks with batch normalisation and ReLU. Input encodes M packet sizes and transmission directions per flow. Optimised with Adam + categorical cross-entropy; hyperparameters selected via grid search with cross-validation.

The 5,000 flows + features configuration achieves the best precision-recall balance (0.841 / 0.842). Lower learning rates (0.0001) consistently outperform larger alternatives. Training runtime: 230–748 s, peak memory 1,107 MB.

08 · Federated Learning

Privacy-Preserving Distributed Detection

The federated learning system is implemented as a client-server architecture in which participating network nodes independently train local autoencoder models on private traffic data and transmit only model parameter updates to a central aggregation server. At no point during training is raw traffic data centralised, rendering this approach applicable to privacy-sensitive telecommunications infrastructure subject to regulatory constraints. The integration of ResNet into the federated pipeline (ResNet-Enhanced variant) is observed to improve F1-score from 89.9% to 93.2%, at the cost of an increase in training time from 2.3 to 3.8 hours per round.

09 · Sequence Analysis

N-gram Sequence Modelling

N-gram analysis is employed to capture temporal dependencies in packet sequences that aggregate flow-level statistics are inherently unable to represent. During the training phase, all n-grams observed in normal traffic are catalogued within a space-efficient Bloom filter structure. At inference time, each incoming packet sequence is assigned an anomaly score proportional to the fraction of constituent n-grams absent from the training catalogue; sequences whose scores exceed a configurable threshold are classified as malicious.

N-gram size Precision (%) Memory (MB) Latency (ms)
n = 2 88.1 23.4 1.13
n = 3 92.3 47.8 1.87
n = 4 94.7 92.5 2.56
n = 5 95.1 163.7 3.42

Detection precision improves monotonically with n; however, memory consumption approximately doubles with each incremental increase in n. For latency-sensitive 5G deployment scenarios, n = 4 is identified as the most practically viable configuration, yielding 94.7% precision at a per-sample inference latency of 2.56 ms with a memory overhead of 92.5 MB.

Feature significance is assessed via permutation importance and information gain across all evaluated classifiers. Results are consistent in their broad conclusions: temporal and size-related features account for the majority of discriminative power. At the individual feature level, outgoing quartile transmission times (outgoing_q1_time, outgoing_q3_time) are identified as the single most important features within the Random Forest model. One-Class SVM assigns greatest weight to burst statistics (max_burst, mean_burst), while Elliptic Envelope exhibits primary dependence on total flow time. This per-algorithm divergence in feature weighting provides empirical justification for the multi-model ensemble approach adopted in this work.

11 · Error Analysis

Misclassification Analysis

Error type Frequency Description Mitigation
Benign anomalies 31.4% Legitimate traffic spikes classified as malicious Adaptive thresholding
Low-volume attacks 28.9% Attacks staying below detection thresholds Enhanced sequence analysis
Protocol ambiguity 23.7% Legitimate traffic on non-standard ports Protocol fingerprinting
Encrypted payload 16.0% Limited visibility into E2E-encrypted traffic Metadata + timing analysis

Dynamic adaptive thresholding is found to outperform static global thresholds across both metrics: precision improves from 88.7% to 94.6% and recall from 92.4% to 91.7%, at a modest per-sample latency increase of 1.08 ms. These results suggest that threshold adaptation constitutes a practical and computationally tractable mitigation strategy for benign anomaly false positives.

12 · Conclusions

Conclusions & Future Work

Multi-algorithm framework. The Random Forest classifier achieves an F1-score of 95.01% under controlled evaluation conditions and 84.74% when generalising to novel attack patterns; the federated ResNet-enhanced autoencoder variant attains 93.2%, demonstrating superior generalisation relative to the baseline federated model.

Synthetic data augmentation. The proposed GAN architecture is shown to reproduce the statistical properties of real 5G traffic with a maximum Shannon entropy deviation of ±3.25%, confirming its suitability for data augmentation without degrading model training integrity.

Privacy-preserving detection. The federated learning framework eliminates raw data centralisation and reduces membership inference attack success from 78.3% to 12.7% — a reduction of 83.8 percentage points — while lowering per-node energy consumption from 1.43 to 0.35 kWh.

Feature importance. Temporal patterns (37.2% relative importance) and packet size distributions (24.8%) are identified as the primary discriminative drivers across classifiers, confirming that meaningful anomaly detection is achievable from metadata-level features in encrypted traffic environments.

Limitations & future directions

  • Encrypted payload opacity. End-to-end encryption removes payload visibility, contributing to 16.0% of total misclassifications; session-level behavioural analysis and side-channel timing represent promising remediation directions.
  • Computational overhead. The federated ResNet-enhanced variant requires 4.8 training hours and 3.7 GB peak memory per node, potentially exceeding the resource envelope of constrained edge elements.
  • Low-volume evasion resilience. Attacks that deliberately remain below detection thresholds account for 28.9% of false negatives; more sophisticated sequence-level analysis and adaptive threshold mechanisms are required to address this attack class.
  • Future work encompasses personalised and asynchronous federated learning protocols, explainability-aware model design, cross-domain knowledge transfer, and extension of the evaluation framework to emerging 6G network architectures.
